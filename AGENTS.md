## Data Flow
- **With server:** `CarContext` ‚Üí `GET /api/cars` ‚Üí SQLite ‚Üí admin can CRUD
- **Without server:** `CarContext` falls back to `src/data/cars.js` (read-only)
- **Seed:** Only runs when DB is empty ‚Äî never deletes existing cars or messages
- **Admin adds car:** POST `/api/cars` ‚Üí file upload ‚Üí saves to SQLite + `uploads/`
- **Contact/Trade-In/Test Drive forms:** POST `/api/messages` ‚Üí stored in SQLite, viewed at `/PDM-admin/messages`
- **Email notifications:** Via Nodemailer + Gmail (requires `EMAIL_PASS` env var)
- **Financing calculator:** pure client-side JS (no backend needed)

## Admin API Flow
- Cars CRUD at `/api/cars` (requires Bearer JWT token from `/api/PDM-admin/login`)
- Login at `/api/PDM-admin/login` returns JWT (valid 7 days)
- Image upload uses `multipart/form-data` with field name `images` (array, max 10)
- First uploaded image is always the primary thumbnail
- Existing images preserved via `existingImages` JSON field in PUT body
- Messages DELETE at `/api/messages/:id` (requires auth)

## Image Handling
- Dealership images: `cdn-thumbor.autodealersdigital.com/unsafe/fit-in/1000x750/...`

- Admin uploads stored at `/uploads/filename` (local filesystem)
- Multiple images per car stored as JSON array in `images` column
- Single `image` column keeps the primary/thumbnail image

## Security
- SQL injection: Safe (all `?` prepared statements with better-sqlite3)
- XSS: Safe (React JSX auto-escapes, no dangerouslySetInnerHTML)
- Auth: JWT + bcrypt, Bearer token required on all write routes
- Passwords: bcrypt(10), never plaintext
- File upload: Extension whitelist (jpg/jpeg/png/webp), 5MB limit, random filenames
- Admin setup: Single-use, disabled after first admin
- Helmet: Security headers (X-Content-Type-Options, X-Frame-Options, etc.)
- CORS: Restricted to Render + localhost origins
- /api/debug: Only in non-production (gated by NODE_ENV)
- JWT secret: From env var only (`JWT_SECRET`), random fallback warns on each restart

## Environment Variables (Render)
- `DB_PATH=/data/data.db` ‚Äî Database path on Persistent Disk
- `JWT_SECRET` ‚Äî JWT signing key (required, set permanent value)
- `ADMIN_PASSWORD` ‚Äî Admin login password (optional, defaults if not set)
- `EMAIL_PASS` ‚Äî Gmail App Password for email notifications (optional)
- `EMAIL_USER` ‚Äî Gmail address for sending (optional, defaults to Powerdrivemotorllc@gmail.com)
- `ADMIN_PHONE` ‚Äî Your WhatsApp number with country code (e.g., `+9647xxxxxxxx`) ‚Äî required for WhatsApp notifications
- `TWILIO_SID` ‚Äî Twilio Account SID (optional, for Twilio WhatsApp API)
- `TWILIO_TOKEN` ‚Äî Twilio Auth Token (optional)
- `TWILIO_WHATSAPP` ‚Äî Twilio WhatsApp sender number (optional)
- `NODE_ENV=production` ‚Äî For production on Render (disables /api/debug)

- Baileys auto-connects on server start ‚Äî auth saved to `wa-auth/` dir (persistent on Render)
- Admin page at `/PDM-admin/whatsapp` to scan QR & check status
- Falls back to generating a `wa.me` link if neither method is configured
- New messages (contact/credit/trade-in/test-drive) auto-send to `ADMIN_PHONE`
- On Render: Baileys auth persists across restarts via Persistent Disk
- Packages: `@whiskeysockets/baileys`, `qrcode`, `pino` (already installed)

## Gotchas
- server/seed.js only seeds when DB is empty ‚Äî safety check prevents re-insert
- `/api/debug` disabled in production ‚Äî gated by NODE_ENV
- backdrop-filter on navbar breaks `position: fixed` children ‚Äî backdrop and side drawer moved outside `<nav>`
- CDN images only accept 4:3 aspect ratio (`fit-in/400x300` works, `fit-in/600x400` returns 403)
- Domain `powerdrivermotor.com` pointed to Render ‚Äî live
- Cloudflare tunnel URL changes every restart (random subdomain) ‚Äî for local dev sharing
- User communicates in Arabic using Latin script (e.g., "shu", "bde", "keef")
- Canvas animation removed (CSS-only effects used instead to avoid lag)
- history.scrollRestoration = "manual" set globally to prevent scroll-to-top issues on navigation
