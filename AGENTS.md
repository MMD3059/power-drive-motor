# Power Drive Motor — Agent Guide

## Commands
```bash
npm run dev       # Frontend dev server (Vite, :5173, proxies /api → :3001)
npm run build     # Production build → dist/
npm run server    # Express backend (:3001, serves dist/ in prod)
node server/index.js  # Run everything together (:3001)
```

## Project Structure
- `server/` — Express + SQLite backend, API at `/api/`
- `src/pages/` — 7 public + 4 admin pages
- `src/data/cars.js` — Static fallback (used when API unavailable)
- `src/contexts/CarContext.jsx` — Fetches from API, falls back to static `cars.js`
- `src/utils/images.js` — `thumbnail(url, size)` for responsive image URLs
- `server/uploads/` — Admin-uploaded car images
- `server/data.db` — SQLite database (autogenerated by seed)
- `src/i18n/` — Native EN/ES translation system (no Google Translate)

## Data Flow
- **With server:** `CarContext` → `GET /api/cars` → SQLite → admin can CRUD
- **Without server:** `CarContext` falls back to `src/data/cars.js` (read-only)
- **Seed:** Only runs when DB is empty — never deletes existing cars or messages
- **Admin adds car:** POST `/api/cars` → file upload → saves to SQLite + `uploads/`
- **Contact/Trade-In/Test Drive forms:** POST `/api/messages` → stored in SQLite, viewed at `/PDM-admin/messages`
- **Email notifications:** Via Nodemailer + Gmail (requires `EMAIL_PASS` env var)
- **Financing calculator:** pure client-side JS (no backend needed)

## Admin API Flow
- Cars CRUD at `/api/cars` (requires Bearer JWT token from `/api/PDM-admin/login`)
- Login at `/api/PDM-admin/login` returns JWT (valid 7 days)
- Image upload uses `multipart/form-data` with field name `images` (array, max 10)
- First uploaded image is always the primary thumbnail
- Existing images preserved via `existingImages` JSON field in PUT body
- Messages DELETE at `/api/messages/:id` (requires auth)

## Image Handling
- Dealership images: `cdn-thumbor.autodealersdigital.com/unsafe/fit-in/1000x750/...`
- Cards use `thumbnail(url, "400x300")` — only 4:3 ratio works (600x400 returns 403)
- Admin uploads stored at `/uploads/filename` (local filesystem)
- Multiple images per car stored as JSON array in `images` column
- Single `image` column keeps the primary/thumbnail image

## Security
- SQL injection: Safe (all `?` prepared statements with better-sqlite3)
- XSS: Safe (React JSX auto-escapes, no dangerouslySetInnerHTML)
- Auth: JWT + bcrypt, Bearer token required on all write routes
- Passwords: bcrypt(10), never plaintext
- File upload: Extension whitelist (jpg/jpeg/png/webp), 5MB limit, random filenames
- Admin setup: Single-use, disabled after first admin
- Helmet: Security headers (X-Content-Type-Options, X-Frame-Options, etc.)
- CORS: Restricted to Render + localhost origins
- /api/debug: Only in non-production (gated by NODE_ENV)
- JWT secret: From env var only (`JWT_SECRET`), random fallback warns on each restart

## Environment Variables (Render)
- `DB_PATH=/data/data.db` — Database path on Persistent Disk
- `JWT_SECRET` — JWT signing key (required, set permanent value)
- `EMAIL_PASS` — Gmail App Password for email notifications (optional)
- `EMAIL_USER` — Gmail address for sending (optional, defaults to Powerdrivemotorllc@gmail.com)
- `NODE_ENV=production` — For production on Render (disables /api/debug)

## Gotchas
- server/seed.js only seeds when DB is empty — safety check prevents re-insert
- `/api/debug` disabled in production — gated by NODE_ENV
- backdrop-filter on navbar breaks `position: fixed` children — backdrop and side drawer moved outside `<nav>`
- CDN images only accept 4:3 aspect ratio (`fit-in/400x300` works, `fit-in/600x400` returns 403)
- Domain parked at GoDaddy — not yet pointed to Render
- Cloudflare tunnel URL changes every restart (random subdomain) — for local dev sharing
- User communicates in Arabic using Latin script (e.g., "shu", "bde", "keef")
- Canvas animation removed (CSS-only effects used instead to avoid lag)
- history.scrollRestoration = "manual" set globally to prevent scroll-to-top issues on navigation
